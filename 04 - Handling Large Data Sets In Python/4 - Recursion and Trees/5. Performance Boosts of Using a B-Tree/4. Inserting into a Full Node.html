We will now be dealing with the case of a full node. A full node is a node that contains len(self.keys) equal to maximum length of 2t1. To make things clear, here's an example between a full node and a non full node.

In the previous screen's example, we dealt with a full node by outright rejecting the key. Obviously we want to make our b-tree function properly so we need to deal with this case. While implementing our solution, we want to keep in mind that our tree should stay balanced and stay in sorted order.

Another requirement is that non-root nodes must also stay within the bounds t1 and 2t1. If we want to stay within those bounds, then notice that if we were to add one more key to a full node, the total keys would be 2t1+1=2t. Suppose we were to split that node into two, then we would have two nodes that have t keys each which is well within our bounds!

However, if we split the nodes into two, we also need to maintain a reference to the newly created node. Therefore, we want to perform the splitting operation within the parent node. Within the parent node, we are able to alter the references in the children list.

To make it easier to write a split method, there is a way to return multiple values in a function call. This helps in the context of the split operation because we want to be able to return two new child nodes and new key. Here's an example of how this can work.

def return_multiple(values):
    left_values = values[:len(values)//2]  #  Grab the left half values.
    right_values = values[len(values)//2:]  #  Grab the right half values.
    new_key = right_values.pop(0)  # Pop the leftmost value.
    return left_values, right_values, new_key  # Return the three values.
left, right, key = return_multiple([1, 2, 3, 4, 5])
print(left, right, key)
