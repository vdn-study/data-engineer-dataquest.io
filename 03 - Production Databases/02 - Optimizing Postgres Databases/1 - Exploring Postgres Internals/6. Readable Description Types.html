Great! Now we have a mapping of all type codes to their type names. Using the type_mappings dictionary, the types are provided without looking them up in the documentation.

Let's put all this together and create our own table descriptions. We want to rewrite the cursor.description attributes from a list of tuples towards something human readable. In the following exercise, we will assemble the output from the previous exercises into this dictionary:

xxxxxxxxxx
 
{
  "homeless_by_coc": {
    "columns": [
      {"name": "id",         "type": "int4",    "internal_size": 4},
      {"name": "year",       "type": "date",    "internal_size": 4},
      {"name": "state",      "type": "char",    "internal_size": 2},
      {"name": "coc_number", "type": "char",    "internal_size": 128},
      {"name": "measures",   "type": "varchar", "internal_size": 64},
      {"name": "count",      "type": "int4",    "internal_size": 4}
    ]        
  }
  "state_info": {
     "columns": [
         {"name": "name",    "type": "varchar",    "internal_size": 14},
         ...
     ]
  }
    ...
}

This dictionary should contain three entries with keys corresponding to the tables names. The above example shows the entry corresponding to the key homeless_by_coc. The value is another dictionary with a single key named columns and whose value is a list. Each entry of that list is a dictionary describing the list. It contains three entries, one for the name, one for the type and one for the internal size.

So for instance, imagine that the dictionary is stored in a variable named readable_description. Then

xxxxxxxxxx
 
readable_description["homeless_by_coc"]["columns"]

is equal to a list containing the dictionary description of each column:

xxxxxxxxxx
 
[
  {"name": "id",         "type": "int4",    "internal_size": 4},
  {"name": "year",       "type": "date",    "internal_size": 4},
  {"name": "state",      "type": "char",    "internal_size": 2},
  {"name": "coc_number", "type": "char",    "internal_size": 128},
  {"name": "measures",   "type": "varchar", "internal_size": 64},
  {"name": "count",      "type": "int4",    "internal_size": 4}
]

Also,

xxxxxxxxxx
 
readable_description["homeless_by_coc"]["columns"][0]

will give the dictionary describing the id column:

xxxxxxxxxx
 
{"name": "id", "type": "int4", "internal_size": 4}

Let's do an exercise to replicate the above dictionary. This is quite challenging, so don't feel frustrated if you don't succeed in the first few attempts. Try to create it little by little and print the intermediate values to see if they match what you expect.

instructions

Variables col_descriptions and type_mappings from the previous screens are available. You will need them to create the inner most dictionary for describing each column.

Recall that col_descriptions is a dictionary with one entry per table. The value for each table is a list of Column objects describing the columns of the table. The type_mappings is another dictionary that given a type code will map to the corresponding type name.

  1. Replicate the dictionary descried above in a variable named readable_description. The dictionary should have:
    • Three entries, one for each table with the key being a string with the table name.
    • Each table name should map to a dictionary with a single string key named columns. The value should be a list containing one dictionary per column in the table.
    • Each dictionary describing a column should contain three keys: name, type and internal_size mapping to the name of the column, name of the data type and internal size of that data type used to represent that column.
