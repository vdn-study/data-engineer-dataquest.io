Wow! That's a lot of good information. Using the ANALYZE option, we have both estimates and actual times (in milliseconds). Furthermore, we are presented with the total execution time at the end.

We can also use the ANALYZE option to record the run times of queries other than SELECT. For example, we can compute the total time it takes for Postgres to add a new column to the state_info table using the following query:

xxxxxxxxxx
 
EXPLAIN ANALYZE
ALTER TABLE state_info 
ADD COLUMN notes text;

As demonstrated above, EXPLAIN ANALYZE can provide us with the runtime of any query. However, recall that the ANALYZE option, as opposed to the default EXPLAIN, will execute the query. Meaning that, for any modifying or destructive queries (DELETE, UPDATE, ALTER, etc.) this will reflect an actual change in the database.

To get the best of both worlds and have analyze leave the table as is, we can call the rollback()method from the connection object after the query. As we have seen before, this will revert any of the changes you made in the current transaction block. 

xxxxxxxxxx
 
import psycopg2
conn = psycopg2.connect(dbname="dq", user="hud_admin", password="abc123")
cur = conn.cursor()
# Modifying change to the database.
cur.execute("""
    EXPLAIN ANALYZE
    ALTER TABLE state_info
    ADD COLUMN notes text;
""")
# Rollback the change.
conn.rollback()
