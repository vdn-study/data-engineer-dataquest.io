On the previous screens, we discussed the cursor.mogrify() method which is what the cursor.execute() method uses to convert Python types into Postgres types when we pass the values as a second argument. We have seen this conversion happen when we mogrified the game data:

 
game_data = (52499790661213, 'Amazing', 'LittleBigPlanet PS Vita', '/games/littlebigplanet-vita/vita-98907', 'PlayStation Vita', 9.0, 'Platformer', 'y', datetime.date(2012, 12, 9))
​
mogrified_values = cur.mogrify("(%s, %s, %s, %s, %s, %s, %s, %s, %s)", game_data)
print(mogrified_values)
xxxxxxxxxx
 
b"(52499790661213, 'Amazing', 'LittleBigPlanet PS Vita', '/games/littlebigplanet-vita/vita-98907', 'PlayStation Vita', 9.0, 'Platformer', 'y', '2012-12-09'::date)"

For example, we can see in the above result that the Python datetime.date object was converted to a Postgres date object '2012-12-09'::date.

This means that if we use the mogrified value, decode it into a string, and append it to a insert statement, we will get the same behavior as if we used the cursor.execute() method with two argument. In particular, doing this is an alternative safe way to covert between types and avoid SQL injections.

In other words, we are saying that the following two ways of inserting are equivalent:

xxxxxxxxxx
 
cur.execute("INSERT INTO ign_reviews VALUES (%s, %s, %s, %s, %s, %s, %s, %s, %s)", game_data)
xxxxxxxxxx
 
mogrified_values = cur.mogrify("(%s, %s, %s, %s, %s, %s, %s, %s, %s)", game_data)
cur.execute("INSERT INTO ign_reviews VALUES" + mogrified_values.decode(conn.encoding))

Using mogrified values, it is possible to insert multiple rows at once into a database table. For the sake of example, let's imagine that we have a table named example_table with three columns, one with integer values and the two others with string values. In Postgres, we can insert multiple rows at one by passing multiple comma separated value tuples to the INSERT command, like so:

xxxxxxxxxx
 
INSERT INTO example_table VALUES (1, 'hello', 'world'), (2, 'data', 'quest'), (3, 'love', 'learning')

To build a single INSERT statement with multiple values, we have to separate each tuple of row values using a comma. Imagine that we want to build a single insert statement for insert the two rows show below.

xxxxxxxxxx
 
first_row = cur.mogrify("(%s, %s, %s, %s, %s, %s, %s, %s, %s)",
    (52499790661213, 
     'Amazing', 
     'LittleBigPlanet PS Vita', 
     '/games/littlebigplanet-vita/vita-98907', 
     'PlayStation Vita', 
     9.0, 
     'Platformer', 
     'y',
     date(2012, 12, 9))
).decode(conn.encoding)
​
second_row = cur.mogrify("(%s, %s, %s, %s, %s, %s, %s, %s, %s)",
    (739958911683746, 
     'Amazing', 
     'LittleBigPlanet PS Vita MSHE', 
     '/games/littlebigplanet-ps-vita-marvel-super-hero-edition/vita-20027059', 
     'PlayStation Vita', 
     9.0, 
     'Platformer', 
     'y', 
     date(2012, 12, 9))
).decode(conn.encoding)

We would build the INSERT statement by first joining all the stringyfied tuples into a single string with commas separating them and then we would concatenate that with an INSERT statement as shown next:

xxxxxxxxxx
 
separated_values = ",".join([first_row, second_row])
cur.execute("INSERT INTO ign_reviews VALUES " + separated_values + ";")
conn.commit()

Let's practice this by building a query for inserting all rows from the ign.csv file at once!

instructions
  1. Connect to the database named dq with user dq.
  2. Load the ign.csv into a list of lists representing the rows. Skip the header row and assign the remaining rows to a list named rows.
  3. Mogrify each row of the list and assign them to a new list named mogrified_rows. Remember that each row has 9 columns.
  4. Decode each mogrified value and assign them to a new list named decoded_rows.
  5. Using ",".join() on the decoded rows list, create an insert string for inserting all rows into the ign_reviews table. Assign the result to a variable named insert_string.
  6. Execute the insert string with the cursor.
  7. Commit your changes and close the connection.
